## 数据分离 + 热加载 + 候选发现（2026-07-15）

### 问题：关键词硬编码在 config.py 中，新增关键词必须改源码 + 重启

### 方案：三模块架构
| 模块 | 职责 |
| --- | --- |
| `keywords_data.json` | 纯 JSON 数据文件，存放全部关键词映射。用户可用任意编辑器修改 |
| `keywords.py` | 加载器：从 JSON 初始化 config 模块全局变量；提供 `reload()` 热加载（更新内存变量，无需重启）；`add_keyword()` 动态写入 JSON |
| `keywords_discovery.py` | 候选发现引擎：当语言=["通用"]（无关键词命中）时，从反引号/CamelCase/UPPER/标题中提取未覆盖的技术术语，按频次展示 |

### 改动文件
- 新建：`keywords_data.json`、`keywords.py`、`keywords_discovery.py`
- 修改：`config.py`（关键词从硬编码 → 从 keywords.py 加载）、`extractor.py`（未命中时调用 discovery）、`ui_app.py`（「热加载关键词」按钮 + 候选面板）

- py_compile 通过
- unittest 12/12 全绿

## NiceGUI 启动问题修复（2026-07-15）

### 根因
NiceGUI 3.14.0 的**全局作用域模式**（`ui.label()` + `ui.run()`）在此版本有兼容问题，返回 500。
必须使用 `@ui.page("/")` 装饰器模式。

### 正确模式
- `ui_app.py`: `@ui.page("/")` 装饰 `create_ui()` 函数
- `main.py`: 只 `from . import ui_app`（触发路由注册）+ `ui.run()`，**不手动调用 `create_ui()`**
- 端口 8080 正常，不是端口问题

### 改动
- `src/main.py`: 去掉 `ui_app.create_ui()` 调用
- `src/ui_app.py`: 恢复 `@ui.page("/")` 装饰器
