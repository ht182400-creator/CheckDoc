## 新建项目：MemoAlign（记忆对齐分析器）
- 目标：将分散在 `**/memory/*.md` 的知识笔记，对齐（归一化）为统一结构化表格，支持 NiceGUI 表头排序、多列筛选、行下钻详情、CSV/Markdown 导出。
- 技术栈：Python + NiceGUI；规则抽取（离线默认，关键词命中）+ 可选 LLM 抽取（OpenAI 兼容，可插拔）。
- 架构：Schema 驱动（config.SCHEMA 为单一真相源，新增字段仅改配置即在 UI 生效）；分层 scanner/extractor/store/ui_app。
- 目录：`docs/`（PM 规范文档 00~07 + README）、`src/`（config/logger/scanner/extractor/llm_client/store/ui_app/main）、`tests/`、`sample/`（即时可测样例）。
- 运行：`pip install -r requirements.txt` → `python -m src.main`（默认 http://127.0.0.1:8080，根目录默认 D:\Work_Area\AI，可在 UI 改）。
- 验证：py_compile 通过；tests/test_pipeline.py 4 项 unittest 全绿；headless 抽取样例 3 文件字段正确。
- 关键设计：增量缓存（path+mtime）加速重扫；单文件异常隔离（记日志+占位记录不中断）；毫秒级 TimedRotatingFileHandler 日志。

## 数据分离 + 热加载 + 候选发现（2026-07-15）

### 问题：关键词硬编码在 config.py 中，新增关键词必须改源码 + 重启

### 方案：三模块架构
| 模块 | 职责 |
| --- | --- |
| `keywords_data.json` | 纯 JSON 数据文件，存放全部关键词映射。用户可用任意编辑器修改 |
| `keywords.py` | 加载器：从 JSON 初始化 config 模块全局变量；提供 `reload()` 热加载（更新内存变量，无需重启）；`add_keyword()` 动态写入 JSON |
| `keywords_discovery.py` | 候选发现引擎：当语言=["通用"]（无关键词命中）时，从反引号/CamelCase/UPPER/标题中提取未覆盖的技术术语，按频次展示 |

### 改动文件
- 新建：`keywords_data.json`、`keywords.py`、`keywords_discovery.py`
- 修改：`config.py`（关键词从硬编码 → 从 keywords.py 加载）、`extractor.py`（未命中时调用 discovery）、`ui_app.py`（「热加载关键词」按钮 + 候选面板）

### 验证
- py_compile 通过
- unittest 12/12 全绿
- 热加载：已知关键词 190 个，`keywords.reload()` 成功刷新
- 候选发现：创建含未覆盖词 `webassembly` 的文件 → 成功检测为候选
- 用户操作路径：发现候选词 → 手动编辑 JSON 或在 UI 面板确认 → 点击「热加载关键词」→ 生效（无需重启）
