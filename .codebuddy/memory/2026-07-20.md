# 2026-07-20 开发日志

## IME 中文检索误搜：第四次修正，改用 JS 桥接（治本）
- 用户反馈：日志显示搜索值仍是 `xiang'dao`（拼音态），不是中文「向导」；且之前「input 事件辅助」的 `IME 合成标志` DEBUG 日志缺失 → 证明 `input` 事件根本没到 Python 端。
- 调研（查 NiceGUI 3.14 源码 `client.py`/`functions/html.py` 确认）：
  - `QInput` 不把 `compositionstart`/`compositionend`/`input` 作为 Python 回调转发。之前的 `compositionstart` 不触发、`input` 触发则搜索失灵，根源在此。
  - `ui.run_javascript(code)` 在 3.14 中 `await` 即返回 JS 结果，且内部 `await client.connected()` 等客户端连接；`add_head_html` 用 `insertAdjacentHTML` 注入 `<script>` 不执行，故必须用 `run_javascript` 跑 JS。`ui.timer(interval, cb, once=True)` 可用。
- 最终方案（`src/_ui_filters.py`）：
  - `IME_COMPOSITION_JS`：客户端 capture 阶段监听原生 `compositionstart/end`，把状态写入 `window.__memoalign_composing`（匹配 `.q-field__native`/INPUT/TEXTAREA），idempotent（guard 变量）。
  - `_inject_ime_composition_js()`：客户端连上后（0.5s `ui.timer once`）注入；失败仅 WARNING 降级纯防抖。
  - `_is_composing()`（async）：`await ui.run_javascript("return window.__memoalign_composing === true;")`，失败返回 False 降级。
  - `_attach_ime_safe_input`：主触发仍用 `update:model-value`（可靠）防抖；`_fire` 改为 async，到点读 JS 标志——合成中推迟、`compositionend` 提交中文后才搜索。**彻底消除中间拼音误搜**。
  - 删除了失效的 `_extract_is_composing`/`_set_composing`/`composing_ref`（版本幽灵）。
- 覆盖：英文/标识符(无合成)、中文经输入法(合成中推迟→提交后搜)、中英文混输、粘贴、清除。
- 验证：`loop/verify.py` verdict ALL PASS；`MEMORY.md` 新增 P7 长期教训（NiceGUI IME 状态必须用 run_javascript 桥接）。
- 请浏览器实测：全文检索框输「向导」应命中且日志应现 `IME 合成中(JS 标志=True)，推迟搜索` 与最终 `归一化后="向导"` 命中；输 `xiangdao` 英文也命中；列 TEXT 筛选框同逻辑。
