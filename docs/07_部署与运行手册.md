# 07 部署与运行手册

> 视角：全栈高级工程师 + 运维

## 1. 环境要求

- Python ≥ 3.10
- 操作系统：Windows / macOS / Linux（路径可任意指定）
- 内存：≥ 512MB（百级文件）

## 2. 安装

```powershell
cd d:\Work_Area\Python\checkdoc
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## 3. 运行

```powershell
python -m src.main
# 打开终端提示的本地地址（默认 http://127.0.0.1:8080）
```

在页面中输入根目录（默认 `D:\Work_Area\AI`），点击「扫描」。

## 4. 可选：启用 LLM 抽取

```powershell
$env:MEMOALIGN_LLM_API_KEY="sk-..."
# 在 src/config.py 将 LLM_ENABLED 设为 True，并可改 LLM_BASE_URL / LLM_MODEL
```

## 5. 缓存与日志

- 缓存：`<项目>/.cache/memory_index.json`（删除即强制全量重扫）。
- 日志：`<项目>/MemoAlign.log`（每日午夜轮转，保留 30 天）。

## 6. 常见问题

| 现象 | 排查 |
| --- | --- |
| 表格为空 | 确认根目录下存在 `memory` 子目录与 `.md` 文件 |
| 扫描很慢 | 首次全抽较慢属正常；后续走缓存 |
| LLM 报错 | 检查密钥/地址；会自动回落规则抽取 |
| 端口占用 | 启动时加 `--port 8081` 或更换 |
| 表格空白 / connection lost | 已治本：显示行瘦身（剔除 `_raw`、TEXT 截断 200 字）+ `main.py` 注入 256MB WebSocket 缓冲；如仍偶发，多为前端缓存旧模块，重启服务即可 |
| 详情打不开 | 检查 `rowClick` 是否按 id 取数（P6 修复）；确认 `S.records` 含完整记录 |
