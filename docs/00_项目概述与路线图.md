# 00 项目概述与路线图

> 视角：产品经理 + 架构师

## 1. 背景与问题

团队在多个项目下通过 `**/memory/*.md` 沉淀了大量"经验/陷阱/规范"笔记（如 `.codebuddy/memory/`）。这些笔记**格式自由、分散、难以检索**：

- 想找"Python 相关的陷阱"要翻遍每个文件；
- 想横向对比"各语言规避方法"没有统一视图；
- 新成员无法快速吸收历史踩坑。

## 2. 产品目标

构建一个本地优先的 **MemoAlign（记忆对齐分析器）**：

1. **扫描**：递归定位根目录下所有 `memory` 子目录中的 `.md` 文件；
2. **对齐**：将每篇自由笔记抽取/归一化为统一字段（内容 / 语言范畴 / 类型 / 规避方法 / 严重度 / 来源 / 标签）；
3. **呈现**：以可交互表格呈现，支持 **表头点击排序 + 多列筛选 + 行下钻详情**；
4. **导出**：一键导出 CSV / Markdown 摘要，便于二次利用；
5. **扩展**：字段 Schema、抽取规则、LLM 后端均可通过配置扩展，无需改业务代码。

## 3. 非目标（边界）

- 不做多用户/云端协作（纯本地工具，数据不出本机）；
- 不内置文档编辑器（只读分析）；
- 不强制依赖 LLM（规则抽取开箱即用，LLM 为可选增强）。

## 4. 成功标准（验收）

| 维度 | 标准 |
| --- | --- |
| 可操作性 | 指定目录 → 一键扫描 → 表格出现，全流程 < 30s（百级文件） |
| 稳定性 | 单文件解析失败不影响整体，记录日志并跳过 |
| 性能 | 增量缓存，二次扫描仅处理变更文件 |
| 扩展性 | 新增字段仅在 `config.SCHEMA` 添加一行即可在 UI 生效 |
| 美观 | 现代卡片式布局、响应式、深浅色友好 |
