## 2. 模块接口

### scanner.scan_iter / scan / scan_memory_md / FileMeta.from_path
通用采集器，产出 `FileMeta`（`path/name/project/mtime/size`）：
- `scan_iter(root, rules=None, max_workers=None)` → 流式生成器，边遍历边 yield（不阻塞，UI 可实时显示进度）。
- `scan(root, rules=None)` → 收集全部 `.md`（通用，不限定 `memory` 目录）。
- `scan_memory_md(root)` → 兼容别名，仅 `MatchRule(dir_name=MEMORY_DIR_NAME)`。
- `FileMeta.from_path(path)` → 由路径构造 `FileMeta`（W2：UI 层构造逻辑下沉至此，project 推导与采集一致）。

### extractor.extract(meta: FileMeta, force_llm: bool = False) -> List[Dict]
按 `SCHEMA` 产出**多条**记录（按 `##` 小节拆分）；优先 LLM（若启用/可用或 `force_llm`），否则规则抽取；当 `language=["通用"]` 自动触发候选发现。`_normalize` 补齐 Schema 默认值并附加内部字段（`_path/_raw/_raw_preview/_mtime/_truncated`）。

### store.load_cache / save_cache / needs_update / get_cached_records / put_all / reset
基于 `path+mtime` 的增量判断：`needs_update` 缺缓存或 mtime 变化才重抽；`put_all` 写缓存时去除 `_raw` 仅留预览；`reset` 清缓存强制全量。

### llm_client.LLMExtractor.extract(text, schema) -> Dict
OpenAI 兼容抽取，超时/解析失败抛异常（由 extractor 兜底到规则）。

### quality_scorer.score_record / score_records / tier_of
- `score_record(rec, llm=False)` / `score_records(recs, llm=False)` → 写入 `quality_score`（0–100）；`llm=True` 且可用走 LLM，否则/H失败回落离线启发式（字段完整度累加）。
- `tier_of(score)` → `高`/`中`/`低`/`未评分`（按 `QUALITY_HIGH_THRESHOLD`/`QUALITY_MID_THRESHOLD`）。

### exporters.build_csv / build_xlsx / build_md / build_pdf / flatten
纯函数导出（与 UI 解耦）：`build_csv` 输出 UTF-8+BOM 字节流；`build_xlsx` 未装 openpyxl 抛 `RuntimeError`（上层回落 CSV）；`build_md` 按类型分组；`build_pdf` 用 reportlab 内置 CID 字体渲染中文；`flatten` 列表/标量统一为可检索字符串。

### record_edit.merge_edit / is_low_score / next_low_score
记录编辑纯逻辑：`merge_edit(rec, edited, schema)` 写回；`is_low_score(rec, threshold)` 判低分或"未评分"；`next_low_score` 取队列首条低分。

### graph_view.build_graph_option(records, mode="treemap") -> dict
依记录大小自动选视图（treemap/分组 treemap/堆叠 bar/network≤100 条），供 `_ui_dialogs._open_graph` 渲染。

### keywords.py —— 关键词管理（热加载）

| 函数 | 说明 |
| --- | --- |
| `reload()` | 从 `keywords_data.json` 重新读取全部关键词，更新 config 模块全局变量。无需重启服务 |
| `add_keyword(section, category, keyword) -> bool` | 向 JSON 文件添加新关键词并同步更新内存 |
| `clear_category(section, category) -> int` | 清空某分类下全部关键词（撤销批量添加） |
| `get_all_known_keywords() -> Set[str]` | 返回当前所有已知关键词的小写集合（供候选发现去重） |

### keywords_discovery.py —— 候选发现

| 函数 | 说明 |
| --- | --- |
| `reset_candidates()` | 清空候选缓存（每次扫描前调用） |
| `discover_from_text(text: str) -> None` | 从文本提取未覆盖术语，累加到缓存 |
| `get_candidates() -> List[Tuple[str, int, str]]` | 返回排序后候选词 [(词, 频次, 来源示例), ...] |
| `pending_for_language(text) -> bool` | 快速判断文本是否可能有未覆盖词（节省计算） |

## 3. UI 事件流

```
_ui_actions.run_scan():
  1. root = 路径输入值（默认 DEFAULT_ROOT）；重入保护（_scanning 锁）
  2. discover.reset_candidates()；force_full（关键词热加载等）→ store.reset()
  3. 阶段一：scanner.scan_iter(root) 流式子线程产出 metas，主协程实时显示"正在扫描：<路径>"
  4. 阶段二：逐文件 extractor.extract(meta)（增量缓存 needs_update 判定）；
     扫描即 quality_scorer.score_records 离线打分；增量扁平化
     S.table.rows.extend(_flatten_one(r))，每 20 文件 S.table.update() 一次（去 O(N²)）
  5. store.save_cache(...)；_ui_filters._build_filter_bar() 重建筛选栏
  6. _ui_filters._update_stats(S.records) 刷新统计；_refresh_candidates() 渲染候选面板
  7. 空数据态硬禁用导出/图谱/修正低分按钮（W11）

「热加载关键词」按钮(_ui_actions._hot_reload_keywords):
  1. keywords.reload() → 更新 config 模块变量
  2. store.reset() + run_scan(force_full=True) 自动重抽，统计立即反映新关键词
  3. toast 提示成功（无需重启）
```

> 统计面板与表格、导出**同源**：三者都基于 `_ui_filters._compute_visible()` 计算的可见完整记录——筛选栏变更触发 `_apply_filters()`（→ `_update_stats` + `S.table.rows = _flatten_records(visible)`）；导出动作直接读 `_compute_visible()` 取完整字段（含原文）。这样检索/筛选后统计、表格、导出始终保持一致。

## 4. 导出格式

导出动作（`_ui_actions._export_*`）统一读 `_compute_visible()` 的**完整记录**（含原文，非瘦身后显示行），再调用 `exporters` 纯函数：

- **CSV**（`exporters.build_csv`）：UTF-8 + BOM 字节流，含全部 Schema 字段 + `_path`/`_truncated`。
- **Excel**（`exporters.build_xlsx`）：依赖 `openpyxl`；未安装时自动回落 CSV 并提示。
- **Markdown**（`exporters.build_md`）：按 `type` 分组输出清单，便于粘贴到文档。
- **PDF**（`exporters.build_pdf`）：依赖 `reportlab`，内置 CID 字体 `STSong-Light` 渲染中文。

## 5. 日志规范（logger.py）

- 格式：`[%(asctime)s.%(msecs)03d] %(levelname)-5s %(name)s:%(lineno)d  %(message)s`
- Handler：控制台 + `TimedRotatingFileHandler`（午夜轮转，`backupCount=30`，后缀 `YYYY-MM-DD`）
- 文件 Handler 初始化失败不影响主流程。
