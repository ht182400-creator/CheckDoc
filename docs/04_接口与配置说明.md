# 04 接口与配置说明

> 视角：代码实现专家 + 全栈工程师

## 1. 配置项（config.py）

| 配置 | 默认值 | 说明 |
| --- | --- | --- |
| `DEFAULT_ROOT` | `D:\Work_Area\AI` | 默认扫描根（UI 可覆盖） |
| `MEMORY_DIR_NAME` | `memory` | 目标子目录名 |
| `MD_EXTENSIONS` | `.md`, `.markdown` | 文件扩展名 |
| `CACHE_FILE` | `<proj>/.cache/memory_index.json` | 增量缓存 |
| `LLM_ENABLED` | `False` | 是否启用 LLM 抽取 |
| `LLM_BASE_URL` | `https://api.openai.com/v1` | 兼容接口地址 |
| `LLM_API_KEY` | `""` | 密钥（建议环境变量注入） |
| `LLM_MODEL` | `gpt-4o-mini` | 模型名 |
| `SCHEMA` | 见 03 | 字段定义（单一真相源） |

> 密钥等敏感配置建议通过环境变量 `MEMOALIGN_LLM_API_KEY` 注入，而非硬编码。

## 2. 模块接口

### scanner.scan_memory_md(root: str) -> List[FileMeta]
递归查找 `**/memory/*.md`，返回文件元信息列表。

### extractor.extract(meta: FileMeta) -> Dict[str, Any]
按 `SCHEMA` 产出记录；优先 LLM（若启用且可用），否则规则抽取。当 language=["通用"] 时自动触发候选发现。

### store.load_cache() / save_cache(data) / needs_update(meta, cache)
基于 `path+mtime` 的增量判断。

### llm_client.LLMExtractor.extract(text, schema) -> Dict
OpenAI 兼容抽取，超时/解析失败抛异常（由 extractor 兜底）。

### keywords.py —— 关键词管理（热加载）

| 函数 | 说明 |
| --- | --- |
| `reload()` | 从 `keywords_data.json` 重新读取全部关键词，更新 config 模块全局变量。无需重启服务 |
| `add_keyword(section, category, keyword) -> bool` | 向 JSON 文件添加新关键词并同步更新内存 |
| `get_all_known_keywords() -> Set[str]` | 返回当前所有已知关键词的小写集合（供候选发现去重） |

### keywords_discovery.py —— 候选发现

| 函数 | 说明 |
| --- | --- |
| `reset_candidates()` | 清空候选缓存（每次扫描前调用） |
| `discover_from_text(text: str) -> None` | 从文本提取未覆盖术语，累加到缓存 |
| `get_candidates() -> List[Tuple[str, int]]` | 返回排序后候选词 [(词, 频次), ...] |

## 3. UI 事件流

```
run_scan():
  1. root = 路径输入值（默认 DEFAULT_ROOT）
  2. discover.reset_candidates()  # 清空旧候选
  3. metas = scanner.scan_memory_md(root)
  4. for meta in metas (带进度):
       if store.needs_update(meta, cache): 
         record = extractor.extract(...)  # 抽取 + 候选发现
       else: record = cache[meta.path]
  5. store.save_cache(...)
  6. discovery.get_candidates() → 渲染候选面板
  7. 渲染统计 + 表格 + 筛选栏

「热加载关键词」按钮:
  1. keywords.reload() → 更新 config 模块变量
  2. toast 提示成功
  3. 下次扫描自动使用新关键词（无需重启）
```

## 4. 导出格式

- **CSV**：`csv.DictWriter`，含全部 Schema 字段。
- **Markdown**：按 `type` 分组输出清单，便于粘贴到文档。

## 5. 日志规范（logger.py）

- 格式：`[%(asctime)s.%(msecs)03d] %(levelname)-5s %(name)s:%(lineno)d  %(message)s`
- Handler：控制台 + `TimedRotatingFileHandler`（午夜轮转，`backupCount=30`，后缀 `YYYY-MM-DD`）
- 文件 Handler 初始化失败不影响主流程。
