# 02 系统架构设计

> 视角：架构师 + 全栈高级工程师

## 1. 架构总览

采用 **分层 + 配置驱动（Schema-Driven）** 架构，UI 与抽取逻辑通过 Schema 解耦，数据与代码通过 JSON 分离，保证可扩展与稳定。

```
┌──────────────────────────────────────────────────────────────┐
│                        UI 层 (NiceGUI)                         │
│  ui_app（组装）· _ui_state（状态）· _ui_helpers（纯函数）         │
│  _ui_filters（筛选/统计）· _ui_dialogs（对话框）· _ui_actions（动作）│
└──────────────┬───────────────────────────────────────────────┘
               │ 调用
┌──────────────▼──────────────────────────────────────────────┐
│                   编排/服务层                                  │
│  扫描 → 抽取 → 缓存 → 候选发现 → 渲染 流水线                     │
└──────────────┬──────────────────────────────────────────────┘
               │ 依赖
   ┌───────────┼────────────┬───────────────────┐
   ▼           ▼            ▼                   ▼
┌────────┐ ┌──────────┐ ┌───────┐  ┌──────────────────────┐
│Scanner │ │Extractor │ │ Store │  │ keywords_discovery.py│
│ 扫描   │ │ 抽取引擎  │ │ 缓存  │  │ 候选关键词发现引擎     │
└────────┘ └────┬─────┘ └───────┘  └──────────────────────┘
                │ 可选                       ▲ 查重
         ┌──────▼──────┐                     │
         │ LLM Client  │               ┌─────┴──────────┐
         └──────┬──────┘               │  keywords.py   │ ← 加载
                │                      │  热加载 / 添加  │
                ▼                      └──────┬─────────┘
         ┌──────────────┐                     │ 读写
         │  config.py   │ ← ← ← ─ ─ ─ ─ ─ ─ ┐
         │ Schema + 常量(缓冲/阈值/颜色)      │
         └──────────────┘            keywords_data.json
                                       (纯 JSON 数据文件)
```

## 2. 模块职责

| 模块 | 职责 | 关键接口 |
| --- | --- | --- |
| `config.py` | 全局常量、字段 Schema、各类常量（缓冲/阈值/颜色/分页/同步间隔） | `SCHEMA`, `LLM_PRESETS`, `TYPE_COLOR_MAP`/`LANG_COLOR_MAP`/`PLATFORM_COLOR_MAP`, `SOCKETIO_MAX_HTTP_BUFFER_SIZE`, `CONTENT_PREVIEW_MAX`, `TABLE_DEFAULT_PAGE_SIZE` |
| `keywords.py` | 关键词加载器：JSON 读写 + `reload()` 热加载 + `add_keyword()`/`clear_category()` | `reload()`, `add_keyword()`, `clear_category()`, `get_all_known_keywords()` |
| `keywords_data.json` | 纯数据文件，存放全部关键词映射（不依赖代码） | 手工编辑或通过 UI 添加 |
| `keywords_discovery.py` | 候选发现引擎：扫描未覆盖术语，生成候选供确认 | `discover_from_text()`, `get_candidates()`, `reset_candidates()`, `pending_for_language()` |
| `logger.py` | 毫秒级日志 + 每日轮转，双输出 | `setup_logger()` / `log` |
| `scanner.py` | 通用流式采集器：递归发现任意目录的 MD，产出 `FileMeta` | `scan_iter(root[, rules])`, `scan(root[, rules])`, `scan_memory_md(root)`（兼容别名）, `FileMeta.from_path(path)` |
| `extractor.py` | 规则抽取 + LLM 兜底 + 候选发现，按 `##` 拆分多记录 | `extract(meta, force_llm=False) -> List[Dict]` |
| `llm_client.py` | OpenAI 兼容 HTTP 抽取（可选） | `LLMExtractor.extract(text, schema)` |
| `store.py` | 基于 mtime 的增量缓存 | `load_cache()`, `save_cache()`, `needs_update()`, `get_cached_records()`, `put_all()`, `reset()` |
| `quality_scorer.py` | 质量评分（LLM 或离线启发式）+ 等级映射 | `score_record()`, `score_records()`, `tier_of()` |
| `exporters.py` | 导出纯函数（CSV/Excel/MD/PDF） | `build_csv()`, `build_xlsx()`, `build_md()`, `build_pdf()`, `flatten()` |
| `record_edit.py` | 记录编辑纯逻辑 | `merge_edit()`, `is_low_score()`, `next_low_score()` |
| `graph_view.py` | 图谱视图（treemap/bar/network 聚合） | `build_graph_option(records, mode)` |
| `sync.py` | 定时增量同步 | `compute_diff()`, `summarize()` |
| `main.py` | 入口：在 `import nicegui` 前 patch `socketio.AsyncServer` 缓冲上限，再启动 | `main()` |
| `ui_app.py` | NiceGUI 页面组装：仅 `@ui.page` 路由 + 布局 + 控件接线（业务逻辑下沉子模块） | `create_ui()`, `_on_row_click()` |
| `_ui_state.py` | UI 共享可变状态单一存放点（各子模块经 `S.xxx` 访问，等价于原 global） | `S.records`, `S.table`, `S.filter_widgets`, … |
| `_ui_helpers.py` | UI 纯函数层（无界面副作用，可单测） | `_build_columns()`, `_flatten_one()`/`_flatten_records()`, `_find_record_by_id()`, `_parse_row_click_id()`, `record_within_time_range()`, `HIGHLIGHT_HEAD_HTML` |
| `_ui_filters.py` | 筛选栏构建 / 过滤逻辑 / 统计面板更新 | `_compute_visible()`, `_apply_filters()`, `_update_stats()`, `_build_filter_bar()`, `_reset_filters()` |
| `_ui_dialogs.py` | 对话框：详情 / 编辑 / 目录浏览 / 图谱 | `_open_detail()`, `_open_edit()`, `_correct_low_scores()`, `_open_dir_browser()`, `_open_graph()` |
| `_ui_actions.py` | 动作/处理器：导出 / LLM 重抽 / 质量评分 / 同步 / 扫描主流程 | `_export_csv/xlsx/md/pdf()`, `_batch_llm_re_extract()`, `_batch_quality_score()`, `_incremental_sync()`, `run_scan()` |

## 3. 关键设计决策

1. **Schema 驱动**：表格列、筛选控件、抽取目标全部由 `config.SCHEMA` 生成。新增字段 = 改配置。
2. **数据分离 + 热加载**：关键词从 `keywords_data.json` 加载，UI 按钮触发 `keywords.reload()` 热更新 config 模块全局变量，无需重启应用。
3. **抽取双引擎**：默认规则抽取（零依赖、离线可用）；可选 LLM 抽取（更准确，可插拔）。二者实现同一接口。
4. **候选发现**：当文件无关键词命中（language="通用"），自动从文本反引号/CamelCase/标题中提取未覆盖的技术术语，展示为候选列表。
5. **异常隔离**：单文件抽取失败 → 记日志 + 生成"解析失败"占位记录，不中断整体。
6. **增量缓存**：`store` 以 `path+mtime` 为键缓存抽取结果，重扫仅处理变更文件，满足性能 NFR。
7. **同步但异步友好**：扫描在 NiceGUI 协程内执行并刷新进度，避免长时间无响应。
8. **UI 分层拆分（P4）**：`ui_app.py` 由约 1380 行拆分为 ~200 行页面组装 + 5 个子模块。`_ui_state` 集中管理共享可变状态（各模块经 `S.xxx` 访问）；`_ui_helpers` 收口无副作用纯函数（可单测）；`_ui_filters`/`_ui_dialogs`/`_ui_actions` 各司筛选/对话框/动作。消除约 30 个模块级全局变量散落与下层逻辑泄漏。
9. **大表性能治本（P5）**：显示行经 `_flatten_one` 剔除 `_raw` 且 TEXT 截断 200 字（`CONTENT_PREVIEW_MAX`），单条体积由 ~37KB 降至 ~1.3KB；详情/导出按需从服务端 `S.records` 取完整记录（`_open_detail` 按 id 取数、导出读 `_compute_visible()`），避免整篇原文经 WebSocket 推送撑爆缓冲。治标：`main.py` 在 `import nicegui` 前 patch `socketio.AsyncServer` 注入 `max_http_buffer_size=256MB`（`SOCKETIO_MAX_HTTP_BUFFER_SIZE`）。
10. **常量收敛（W3）**：颜色/阈值/分页/同步间隔/缓冲上限等魔法数字收口到 `config.py` 命名常量（如 `QUALITY_HIGH_THRESHOLD`、`AUTOSYNC_INTERVALS`、`DIR_BROWSE_MAX`）。
11. **行点击稳健（P6）**：`rowClick` 显式 `args=["row"]`，解析收口 `_ui_helpers._parse_row_click_id`（dict→id，其它防御返回 None），避免对 Quasar 整包列表 `[evt,row,pageIndex]` 调 `.get` 崩溃。

## 4. 扩展性要点

- **加字段**：`config.SCHEMA` 追加 `FieldDef`，UI 自动出现新列与筛选器（列由 `_ui_helpers._build_columns` 生成，筛选器由 `_ui_filters._build_filter_bar` 生成）。
- **换抽取逻辑**：实现新的 `Extractor` 并在 `extractor.py` 注册。
- **换 LLM 厂商**：改 `LLM_PRESETS` / `LLM_BASE_URL` 即可（OpenAI 兼容协议）；UI 提供"后端"下拉切换自动填充。
- **LLM 抽取 UI 化**（P1-2）：`_ui_actions` 中一键开关 + 批量重抽按钮 + 连接状态指示。
- **图谱视图**（P2-1）：`graph_view.py` 的 `build_graph_option(records, mode)` 按语言×类型聚合（treemap/bar/network）。
- **加导出格式**：在 `exporters.py` 增 `build_*` 纯函数（`_ui_actions._export_*` 调用），与 UI 解耦。
- **定时同步**（P2-3）：`sync.py` 的 `compute_diff`/`summarize` + `_ui_actions._incremental_sync`/`_toggle_autosync`。
- **加统计维度**：改 `_ui_filters._update_stats`（指标卡 + 分类 chips + 语言×类型交叉统计）。
- **加筛选器**：由 `SCHEMA.filterable` 自动生成；自定义筛选在 `_ui_filters._build_filter_bar` 追加。

> 扩展路线详见 [08 综合评审报告](08_综合评审报告.md) 统一优先级总表。

## 5. 稳定性与性能

- 文件 IO / 网络（LLM）全部 try-catch，日志含 `traceback.format_exc()`。
- 大目录扫描使用生成器 + 进度回调，避免一次性载入内存。
- 缓存文件损坏时自动重建（catch JSONDecodeError）。
