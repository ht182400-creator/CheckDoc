## 3. 阻断项详述（主代理已读码核实）

### B1【功能·已核实】详情对话框"编辑"按钮失效
- **位置**：`src/ui_app.py:1372`
  ```python
  ui.button("✏️ 编辑", on_click=lambda: _open_edit(row)).props("outline")
  ```
- **根因**：`detail_dialog` 在 `create_ui()` 中**只构建一次**；其中 `row` 是自由变量，在 `create_ui` 局部作用域与全局作用域（全局变量表 ui_app.py:46-58 及 create_ui 的 global 声明均无 `row`）均未定义 → 点击编辑触发 `NameError: name 'row' is not defined`，或在某些重构路径下打开错误（首次/残留）记录，**写回/重算的是错误记录，造成数据污染**。
- **修复方案**：
  1. 新增模块级变量 `current_detail_row = None`；
  2. 在 `_open_detail(row)` 内 `global current_detail_row; current_detail_row = row`；
  3. 按钮改为 `on_click=lambda: _open_edit(current_detail_row)`（点击时取最后查看的记录，正是用户当前查看的）。
- **影响**：详情页"✏️ 编辑"功能完全不可用，属高优先级阻断。

### B2【功能·已核实】PDF 导出遇 HTML 特殊字符崩溃
- **位置**：`src/exporters.py:137-139`（`Paragraph(f"• {content}"…)` 等）+ `src/ui_app.py:483` 的 `except Exception` 捕获。
- **根因**：reportlab `Paragraph` 将文本按 XML/mini-HTML 解析。当记录 `content`/`avoidance`/`language` 含 `&`、`<`、`>`（如 "a & b"、"if x < 5"、"<tag>"）时抛 `SAXParseException`，被 `ui_app.py:483` 捕获为 `ui.notify("PDF 导出失败: …")`，**导出功能对该类记录直接失败**。
- **修复方案**：构造 `Paragraph` 前对文本做 XML 转义——
  ```python
  from xml.sax.saxutils import escape
  content = escape(flatten(r.get("content")) or "（无内容）")
  # lang / sev / av / q_text 同样 escape
  ```
- **补充测试**：在 `tests/test_exporters.py` 增加 `content="a & b <tag> x > y"` 的用例，断言 `build_pdf` 不抛异常且返回合法 PDF。
- **影响**：含常见符号的笔记（工程类笔记极易命中）PDF 导出失败。

### B3【工程化·已核实】推送器非 UTF-8 文件致整轮推送失败
- **位置**：`_gh_push.py:157`
  ```python
  with open(abs_path, "r", encoding="utf-8") as f:
      content = f.read()
  ```
- **根因**：对所有待推送文件强制 UTF-8 读取。若仓库内存在 GBK/其他编码文件（如诊断日志、未排除的 `.txt`）未被 `_should_exclude` 过滤，则抛 `UnicodeDecodeError`，`main()` 捕获后 `sys.exit(1)`——**整轮提交+打 tag 全部失败**。此前已因此踩过坑（见 BACKLOG 风险区）。
- **修复方案**（已实施，采用稳健的编码探测回落）：
  - 新增 `_read_text(abs_path)`：先以二进制读取，再 `raw.decode("utf-8")`；若抛 `UnicodeDecodeError` 则 `log.warning` 并回落 `raw.decode("gb18030")`（GBK/GB2312 超集，可覆盖常见中文编码）。这样既不会因单个 GBK 文件中断整轮，也避免静默 `errors="replace"` 把源码推成乱码。
  - 另：非 UTF-8 诊断文件（如 `_server.log`、`_test_out.txt`）已通过 `ROOT_TEMP` / `TESTS_TEMP_PREFIX` 等排除规则不进推送清单，进一步降低触发概率。
- **影响**：任何非 UTF-8 文件都会阻断发布流程，是发布链路的单点故障。
