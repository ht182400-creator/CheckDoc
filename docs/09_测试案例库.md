## 日志规范

- 测试运行日志：`tests/test_run.log`（每次运行追加）
- 日志格式：`[时间.毫秒] PASS/FAIL/ERROR test_id`
- 失败案例含完整 traceback
- JSON Runner 使用 `_LoggedRunner` 类自动记录

## 新增用例流程

**JSON 数据驱动线：**
1. 编辑 `tests/test_data.json`，在对应模块数组中追加一行
2. 编辑本文件 (§09)，追加用例表格行
3. 运行 `python -m unittest tests.test_runner`
4. 检查 `tests/test_run.log` 确认 PASS

**Python 单测套件线（新功能默认走这条）：**
1. 在对应 `tests/test_*.py` 套件新增 `def test_*`（类名 `TestXxx`）
2. 在 `loop/verify.py` 的套件列表中登记该模块
3. **必须同步本文件 §13.N 追加对应用例行**（ID 用模块前缀：Q/SY/HL/EXP/RE…）
4. 运行 `python loop/verify.py` 确认全绿
