# 06 质量保障与测试方案

> 视角：数据分析师 + 代码实现专家

## 1. 测试分层

| 层级 | 对象 | 方法 |
| --- | --- | --- |
| 单元 | scanner / extractor / store | pytest，构造临时目录与样例 md |
| 集成 | 扫描→抽取→缓存→渲染 | 用样例根目录跑通流水线 |
| 边界 | 空目录、无 memory 目录、超大文件、坏缓存 | 断言不崩溃、日志正确 |
| 异常 | 文件无读权限、编码错误、LLM 超时 | 断言隔离与回落 |

## 2. 关键用例

1. `scan_memory_md`：构造 `root/A/memory/x.md`、`root/B/memory/y.md` → 返回 2 条；构造无 memory 目录 → 返回 0。
2. `extract`（规则）：含 "python"+"陷阱"+"规避" 的样例 → `language=Python, type=陷阱, avoidance` 命中。
3. `store`：同 mtime 命中缓存；改 mtime 触发重抽；坏 JSON 自动重建。
4. UI：模拟点击扫描 → 表格行数 = 文件数；筛选"Python" → 仅 Python 行；表头排序 → 顺序变化。
5. 导出：CSV 行数 = 记录数，字段完整。

## 3. 性能基准

- 100 个各 5KB md：首扫 < 30s，重扫（全缓存）< 3s。
- 单文件异常不影响整体，日志含 traceback。

## 4. 质量门禁

- `python -m py_compile src/*.py` 通过；
- `pytest` 全绿；
- 日志无 ERROR 级（除预期异常）。
