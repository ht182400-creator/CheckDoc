# 06 质量保障与测试方案

> 视角：数据分析师 + 代码实现专家

## 1. 测试分层

| 层级 | 对象 | 方法 |
| --- | --- | --- |
| 单元 | `_ui_helpers`（纯函数：列生成/扁平化/查找/id 解析/时间范围）、`record_edit`、`quality_scorer`、`exporters`、`graph_view`、`scanner`（`FileMeta`/`MatchRule`/`scan_iter`） | `unittest`（标准库，无需 pytest），构造临时目录与样例 md |
| 集成 | 扫描→抽取→缓存→渲染→导出 | 用样例根目录跑通流水线 |
| 边界 | 空目录、无 `.md` 文件、超大文件、坏缓存、PDF 超大文本 | 断言不崩溃、日志正确 |
| 异常 | 文件无读权限、编码错误、LLM 超时、rowClick 误传列表 | 断言隔离与回落（`_parse_row_click_id` 防御） |

## 2. 关键用例

1. `scan`（默认通用）：构造 `root/A/memory/x.md`、`root/B/notes/y.md`（普通目录）、`root/C/doc.md` → 默认返回全部 `.md`；构造无 `.md` 文件 → 返回 0。注：`scan_memory_md` 为兼容别名（仅认 `memory` 目录），通用默认场景使用 `scan`/`scan_iter`。
2. `extract`（规则）：含 "python"+"陷阱"+"规避" 的样例 → `language=Python, type=陷阱, avoidance` 命中；`extract` 按 `##` 返回多条记录。
3. `store`：同 mtime 命中缓存；改 mtime 触发重抽；坏 JSON 自动重建。
4. `_ui_helpers` 纯函数：列生成、扁平化（剔除 `_raw`/截断）、`_find_record_by_id`、`_parse_row_click_id`（dict→id；列表/None→None 防御）、`record_within_time_range` —— 直接单测，覆盖正常值/边界/异常。
5. UI 集成（冒烟）：`loop/verify.py` 的 `ui_import` 导入 `ui_app` 成功（确认拆分后无循环依赖）；模拟点击扫描 → 表格行数 = 文件数；筛选"Python" → 仅 Python 行；表头排序 → 顺序变化。
6. 导出：CSV/Excel/MD/PDF 行数 = 可见记录数（`_compute_visible()`），字段完整（含原文）；未装 openpyxl 时 Excel 回落 CSV。

## 3. 性能基准

- 100 个各 5KB md：首扫 < 30s，重扫（全缓存）< 3s。
- 单文件异常不影响整体，日志含 traceback。

## 4. 质量门禁

- `python -m py_compile src/*.py` 通过（编译无错）；
- `python -m unittest discover -s tests` 全绿（标准库 unittest，9 套件：pipeline/quality/sync/highlight/exporters/record_edit/ui_helpers/scanner/runner）；
- `python loop/verify.py` 统一门禁：编译 + `ui_import` 导入冒烟 + 案例库（`docs/09`）静态检查（用例数与 JSON 一致）+ 9 套件全绿，失败 `exit 1`；
- 日志无 ERROR 级（除预期异常）。
