# 01 需求规格说明书（SRS）

> 视角：产品经理 + 数据分析师

## 1. 用户故事

| 编号 | 角色 | 故事 | 验收 |
| --- | --- | --- | --- |
| US-1 | 分析师 | 作为用户，我想指定任意根目录，一键扫描出所有 `memory` 下的 MD | 默认 `D:\Work_Area\AI`，可改；显示扫描进度与文件数 |
| US-2 | 分析师 | 我想把每篇笔记对齐成统一表格字段 | 表格含 内容/语言范畴/类型/规避方法/严重度/来源/标签 |
| US-3 | 分析师 | 我想点击表头排序 | 点击列头在 升/降 间切换，带排序指示 |
| US-4 | 分析师 | 我想按列筛选内容 | 文本列支持关键字；枚举列（类型/语言/严重度）支持下拉多选 |
| US-5 | 分析师 | 我想点开某行看原文与元数据 | 行点击弹出详情（全文 Markdown + 路径 + 抽取字段） |
| US-6 | 分析师 | 我想导出结果 | 导出 CSV 与 Markdown 摘要 |
| US-7 | 工程师 | 我想新增/调整字段而不改 UI 代码 | 仅在 `config.SCHEMA` 配置即可 |
| US-8 | 工程师 | 我想在没网/没密钥时也能用 | 规则抽取为默认，LLM 为可选 |

## 2. 功能需求

- **FR-1 扫描**：递归查找根目录下所有名为 `memory` 的目录，收集其 `.md`/`.markdown` 文件。
- **FR-2 抽取**：对每个文件产出结构化记录（Schema 见 03）。
- **FR-3 表格**：渲染为可排序、可筛选表格；列来自 Schema，自动生成。
- **FR-4 统计**：顶部展示总数、按类型/语言分布。
- **FR-5 详情**：行下钻展示原文与元数据。
- **FR-6 导出**：CSV（全字段）、Markdown（分组摘要）。
- **FR-7 缓存**：基于文件 mtime 的增量缓存，加速重扫。
- **FR-8 配置**：根目录、LLM 开关/地址/密钥/模型、字段 Schema 均可配置。

## 3. 非功能需求

- **NFR-1 性能**：百级文件首扫 < 30s；重扫 < 3s（缓存命中）。
- **NFR-2 稳定**：单文件异常隔离，整体不中断；异常写入日志。
- **NFR-3 扩展**：Schema 驱动，新增字段零代码改动 UI。
- **NFR-4 可观测**：毫秒级日志 + 控制台 + 文件双输出。
- **NFR-5 易用**：UI 操作 < 3 步到达核心结果。

## 4. 验收用例（示例）

1. 给定 `D:\Work_Area\AI` 含 `projA/memory/a.md`、`projB/memory/b.md` → 扫描得 2 行。
2. 点击"类型"表头 → 按类型排序。
3. 在"语言范畴"筛选选 Python → 仅显示 Python 相关行。
4. 点击某行 → 弹出该 md 全文与字段。
5. 点击导出 CSV → 下载含全部字段的表格。
