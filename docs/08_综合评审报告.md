## 二、核心缺陷：Shell / PowerShell / DOS / 命令行问题无力覆盖

### 2.1 问题现象

用户的实际记忆笔记大量涉及 Shell 相关问题，例如：

```
"构建前必须清 dist/renderer 缓存（rmdir /s /q dist\renderer）"
"CMD copy /Y 在 pnpm 长路径下静默失败"
"node-gyp rebuild 输出为 better_sqlite3.node"
"构建链：core tsup → server tsup → desktop tsup → rmdir → vite build → electron-builder --win"
"上传使用 _release.mjs (PowerShell版)"
```

当前分析器会将所有这些归类为：

- **语言范畴** → `["通用"]`（因为 `LANGUAGE_KEYWORDS` 中根本没有 `powershell`、`cmd`、`bat`、`shell`、`bash` 等关键词）
- **类型** → `"其他"` 或 `"问题"`（因为 TYPE_KEYWORDS 中没有"命令行"、"环境配置"、"兼容性"等分类）

**后果**：用户无法筛选出"所有与 PowerShell 相关的问题"，也无法统计"Shell 环境问题占比多少"。分析器在最关键的使用场景上失效了。

### 2.2 根因分析

#### 根因 A：`LANGUAGE_KEYWORDS` 缺少整个 Shell/CLI 领域

```python
# 当前只有 4 个语言类别
LANGUAGE_KEYWORDS: dict = {
    "Python": [...],
    "JavaScript/TS": [...],
    "Go": [...],
    "Rust": [...],
}
# 缺失：Shell/Bash, PowerShell, Batch/CMD, Docker, 跨平台, 操作系统
```

**缺失的关键词**（来自用户实际笔记中的模式）：

| 缺失语言范畴 | 最少应含关键词 |
| --- | --- |
| **Shell/Bash** | bash, zsh, sh, shell, .sh, chmod, awk, sed, grep, wget, curl, shebang |
| **PowerShell** | powershell, pwsh, ps1, psm1, cmdlet, get-childitem, set-executionpolicy, Out-File |
| **Batch/CMD** | cmd, batch, bat, cmd.exe, .bat, .cmd, dos, errorlevel, setlocal, chcp, rmdir, xcopy |
| **环境/平台** | 环境变量, PATH, env, choco, winget, scoop, brew, apt, yum, 注册表, registry |

#### 根因 B：`TYPE_KEYWORDS` 缺少命令行专属类型

```python
# 当前缺少
# - "环境配置": 安装、卸载、环境变量、PATH配置、权限问题
# - "命令行": 执行失败、参数错误、转义问题、shell差异
# - "兼容性": Windows vs Linux、跨平台、编码问题、路径分隔符
```

Shell 问题的典型模式无法匹配到合适类型。

#### 根因 C：`AVOIDANCE_HEADERS` 缺少 Shell 场景常用标题

```python
# 当前
AVOIDANCE_HEADERS: List[str] = ["规避", "避免", "建议", "注意", "解法", "解决", "修复方法"]

# 缺失 Shell 笔记常出现的标题词
# "解决方法"、"替代命令"、"workaround"、"临时方案"、"改用"、"正确命令"
```

### 2.3 影响量化

用一个简单实验验证（基于用户 `.codebuddy/memory/MEMORY.md` 中实际笔记）：

| 笔记片段 | 当前语言 | 当前类型 | 期望语言 | 期望类型 |
| --- | --- | --- | --- | --- |
| `"CMD copy /Y 在 pnpm 长路径下静默失败"` | 通用 | 问题 | Batch/CMD + JS/TS | 命令行 |
| `"rmdir /s /q dist\renderer"` | 通用 | 问题 | Batch/CMD | 环境配置 |
| `"_release.mjs (PowerShell版)"` | 通用 | 发布 | PowerShell + JS/TS | 发布 |
| `"electron-builder --win"` | JavaScript/TS | 发布 | JavaScript/TS + Shell | 发布 + 命令行 |

**预期命中 vs 实际命中 ≈ 75% 信息丢失**。用户说"没有分析和统计"是完全准确的。

## 三、其他发现的问题

### 3.1 架构层

| 问题 | 严重度 | 说明 |
| --- | --- | --- |
| 缺少**平台维度**字段 | 高 | 同一问题是 Windows-only 还是跨平台？无法筛选 |
| 缓存膨胀 | 中 | `_raw` 全文存入缓存，百篇笔记可达 10MB+ JSON |
| 缺少**关联目标** | 中 | 一篇笔记常涉及多个语言/平台（如 "PowerShell调用Python"），但目前语言字段仅靠关键词，无法解析关联关系 |

### 3.2 代码层

| 问题 | 严重度 | 文件 | 说明 |
| --- | --- | --- | --- |
| **重入保护缺失** | 中 | ui_app.py:199 | 扫描期间再次点击「扫描」，两个协程同时操作 `records`/`cache`，会导致数据错乱 |
| **scan_btn 未 disable** | 低 | ui_app.py:261 | 文档（05_UI）说了会禁用但代码未实现 |
| `_read_text` 超过 `READ_CHUNK_LIMIT` 只截断不提示 | 低 | extractor.py:23 | 静默截断，用户不知道数据不完整 |
| `_detect_type` 优先级策略有风险 | 中 | extractor.py:50-56 | "注意"在陷阱和问题中都是关键词但陷阱**优先级更高**，一篇只说"注意Vite缓存"的笔记可能被错归为"陷阱" |

### 3.3 数据算法层

| 问题 | 严重度 | 说明 |
| --- | --- | --- |
| **关键词完全依赖分词顺序** | 中 | `"pip install"` 在 `.lower()` 后匹配 `"python"` → 对；但 `"shell 中调用 node"` → 可能同时命中多种语言→归为 JS/TS，Shell 丢失 |
| **规避方法提取仅限小节** | 中 | Shell 问题常常在正文中散列，如 `"解决方法: 执行 --verbose 看详细日志"` → 当前 `_extract_section` 无法提取 |
| **统计维度单薄** | 低 | 仅按类型/语言，无交叉分析（如"语言×类型"矩阵、"平台×严重度"热力图） |

### 3.4 UI/UX 层

| 问题 | 严重度 | 说明 |
| --- | --- | --- |
| 缺少**全局搜索** | 中 | 用户想搜"PowerShell"关键词需在每个列筛选输入，不便 |
| 缺少**排序指示器** | 低 | 表头可排序但没有 ▲/▼ 图标，Quasar 默认支持但需要确认 |
| 表格筛选栏太长 | 低 | 8 个字段全放一行，在小屏幕上拥挤 |
| 导出 CSV 不含 `_path` | 低 | 用户下载后找不到原始文件路径 |
| 缺少 `.gitignore` | 中 | `.cache/`、`__pycache__/`、`*.log`、`.venv/` 均未忽略 |

### 3.5 安全与健壮性

| 问题 | 严重度 | 说明 |
| --- | --- | --- |
| **日志文件放在项目根** | 低 | `MemoAlign.log` 在项目根，多人使用时写冲突 |
| 路径泄漏风险 | 低 | 导出 CSV 含来源字段，可能含敏感路径信息 |
| LLM API Key 未脱敏日志 | 低 | 若启用 LLM 且 API 调用失败，错误响应可能含密钥（虽概率低但应防范） |

### 3.6 测试层

| 问题 | 严重度 | 说明 |
| --- | --- | --- |
| **零个 Shell/PowerShell 测试用例** | 高 | 无任何测试验证 Shell 场景 |
| 缺少并发扫描测试 | 中 | 重入保护缺失，是隐患 |
| 缺少大文件测试 | 低 | 超过 2MB 的 MD 文件未测截断行为 |

## 六、总结

**优点**：Schema 驱动架构、抽取双引擎、增量缓存、异常隔离、日志规范——这些设计决策是成熟的、可扩展的。测试全绿、文档完整，项目治理规范。

**核心缺口**：`LANGUAGE_KEYWORDS` / `TYPE_KEYWORDS` 的覆盖范围偏窄，**完全缺失 Shell/PowerShell/Cmd 语言范畴和命令行/环境配置/兼容性类型**。这是用户反馈"没有分析和统计 PowerShell/DOS 问题"的直接根因——约 75% 的相关笔记被归为"通用—其他"，无法被筛选和统计。

**建议行动**：立即执行 P0 补丁（改动仅 config.py + extractor.py，约 2.5 小时），即可让分析器覆盖用户的真实使用场景。
