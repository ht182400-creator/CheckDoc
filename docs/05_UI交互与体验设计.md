# 05 UI 交互与体验设计

> 视角：UI 专家 + 产品经理

## 1. 设计原则

- **信息密度适中**：顶部统计 → 中部筛选 → 主表格 → 详情弹窗。
- **操作 ≤ 3 步到结果**：输入路径 → 扫描 → 看表。
- **反馈即时**：扫描进度条、空状态提示、异常 toast。
- **美观现代**：卡片阴影、圆角、克制配色、深浅色友好。
- **UI 分层**（见 02 架构设计）：`ui_app.py` 仅负责 `@ui.page` 页面组装与控件接线；筛选/统计（`_ui_filters`）、对话框（`_ui_dialogs`）、动作（`_ui_actions`）、共享状态（`_ui_state`）、纯函数（`_ui_helpers`）各自成模块，纯逻辑可单测。

## 2. 布局

```
┌──────────────────────────────────────────────┐
│  标题栏：MemoAlign · 记忆对齐分析器            │

└──────────────────────────────────────────────┘
```

## 3. 交互细节

| 交互 | 行为 |
| --- | --- |
| 点击表头 | 在该列 升→降→无 循环排序，显示 ▲/▼ 指示 |
| 文本筛选 | 输入即过滤（debounce 300ms），跨该列子串匹配 |
| 枚举筛选 | 下拉多选，命中任一即显示 |
| 行点击 | 弹出详情对话框：按记录 **id** 从服务端 `S.records` 取完整记录，渲染全文 Markdown + 元数据 + 字段（显示行已瘦身，详情按需现取原文，避免大表白屏） |
| 重置筛选 | 清空所有筛选与排序 |
| 导出 | 浏览器下载文件，toast 提示成功 |

## 4. 可访问性与健壮性

- 表头用 `sortable` 明确标识可排序列；
- 空结果展示友好占位（"未匹配到记录"）；
- 扫描中禁用重复点击（重入保护 `_scanning` 锁），避免并发混乱；
- 长文本列（内容/规避方法）限制最大宽度 + 省略，详情中展开；
- 表格显示行已**瘦身**（`_flatten_one` 剔除 `_raw`、TEXT 截断 200 字），单条体积由 ~37KB 降至 ~1.3KB，配合 `main.py` 注入 256MB WebSocket 缓冲，2 万条大表不再白屏（治本 + 治标）；
- 详情按记录 **id** 从服务端 `S.records` 现取完整原文（含 `#detail-body` 内代码块 highlight.js 高亮，离线优雅降级为纯文本）；
- 空数据态**硬禁用**导出/图谱/修正低分等依赖数据的按钮（W11），扫描出数据后自动启用。

## 5. 视觉令牌（建议）

- 主色 `#3B82F6`（蓝），危险 `#EF4444`，背景 `#F8FAFC`/深色 `#0F172A`；
- 圆角 `8px`，卡片阴影 `0 1px 3px rgba(0,0,0,.1)`；
- 字体：系统无衬线，表格 `13px`。
